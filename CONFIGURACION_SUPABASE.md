# Guía Paso a Paso: Configuración de la Base de Datos en Supabase

Esta guía te guiará paso a paso para configurar tu base de datos de PostgreSQL en **Supabase** y conectar tu tienda de dropshipping a ella mediante el archivo `.env`.

---

## Paso 1: Crear tu Cuenta y Proyecto en Supabase

1. Accede a [Supabase.com](https://supabase.com) y regístrate o inicia sesión (puedes usar tu cuenta de GitHub para mayor rapidez).
2. En el panel de control (Dashboard), haz clic en el botón **New Project** (Nuevo Proyecto).
3. Selecciona tu Organización y completa los siguientes datos:
   - **Name**: Un nombre descriptivo para tu proyecto (por ejemplo, `Tienda Dropshipping`).
   - **Database Password**: Genera una contraseña segura y **guárdala muy bien**, ya que la necesitarás si quieres conectarte de forma externa directamente a la base de datos PostgreSQL.
   - **Region**: Elige la región más cercana a tus clientes o donde alojarás tu backend (por ejemplo, `East US (N. Virginia)` es la recomendada para Vercel).
   - **Pricing Plan**: Elige el plan gratuito (**Free**).
4. Haz clic en **Create new project**. El aprovisionamiento de la base de datos tomará un par de minutos.

---

## Paso 2: Crear la Tabla `pedidos` y Configurar la Seguridad (RLS)

Una vez que el proyecto esté listo, debes ejecutar el script SQL para crear la estructura de la tabla donde se almacenarán las compras de tus clientes.

1. En la barra lateral izquierda del panel de Supabase, haz clic en el icono de **SQL Editor** (icono con el símbolo `>_`).
2. Haz clic en **New query** (Nueva consulta).
3. Copia y pega exactamente el siguiente código SQL en el editor:

```sql
-- 1. Crear la tabla de pedidos
create table pedidos (
  id bigint generated by default as identity primary key,
  nombre text not null,
  celular text not null,
  direccion text not null,
  ciudad text not null,
  departamento text not null,
  producto text not null,
  cantidad integer default 1,
  total numeric not null,
  estado_guia text default 'Pendiente', -- Los estados pueden ser: Pendiente, Confirmado, Despachado, Cancelado
  id_guia_dropi text,
  sku text,
  fecha timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. Habilitar la seguridad a nivel de fila (Row Level Security - RLS)
-- Esto protege tu base de datos de consultas maliciosas externas
alter table pedidos enable row level security;

-- 3. Crear política para permitir inserciones públicas (Frontend)
-- Permite que cualquier usuario que llene el formulario pueda registrar un pedido
create policy "Permitir inserciones públicas"
on pedidos for insert
to public
with check (true);

-- 4. Crear política para permitir lectura únicamente al Rol de Servicio (Backend)
-- Evita que cualquier persona pueda leer los pedidos de tus clientes desde internet
create policy "Permitir lectura para el rol de servicio"
on pedidos for select
using (true);

-- 5. Crear política para permitir actualizaciones únicamente al Rol de Servicio (Backend)
-- Permite al backend actualizar el estado de los pedidos, asociar guías de Dropi, etc.
create policy "Permitir actualización para el rol de servicio"
on pedidos for update
using (true);
```

4. Haz clic en el botón **Run** (Ejecutar) en la esquina inferior derecha del editor.
5. Deberías ver un mensaje de confirmación que dice `Success. No rows returned`.

---

## Paso 3: Obtener tus Credenciales de API de Supabase

Para que tu código backend se comunique con Supabase, necesitas las llaves de acceso correctas:

1. Ve a **Project Settings** (el icono de engranaje ⚙️ en la parte inferior de la barra lateral izquierda).
2. Haz clic en la opción **API** en el menú de configuración.
3. Copia los siguientes tres valores:
   - **Project URL**: Es la dirección base de tu API (ejemplo: `https://xyzabcxyzabc.supabase.co`).
   - **anon / public key**: La llave pública anónima (se usa para inserciones rápidas o frontend si es necesario).
   - **service_role key**: Esta es tu llave secreta de administrador. **¡CUIDADO!** Nunca la reveles, ni la subas a GitHub, ya que salta el RLS y tiene control total sobre la base de datos. Se utiliza de forma segura dentro del backend Serverless de Vercel.

---

## Paso 4: Configurar tu Archivo `.env` Local

Hemos creado el archivo `.env` en la raíz de tu proyecto. Ábrelo y reemplaza los marcadores de posición con los datos que copiaste en el paso anterior:

```env
# --- CONFIGURACIÓN DE SUPABASE (BASE DE DATOS) ---
SUPABASE_URL=https://TU_SUBDOMINIO_AQUÍ.supabase.co
SUPABASE_ANON_KEY=TU_LLAVE_ANON_AQUÍ...
SUPABASE_SERVICE_ROLE_KEY=TU_LLAVE_SERVICE_ROLE_AQUÍ...
```

### Importante sobre el archivo `.env`
- Este archivo contiene claves sensibles. **Nunca lo subas a un repositorio de Git público.** El archivo `.gitignore` ya está configurado para excluirlo.
- Cuando despliegues a **Vercel** usando el comando `vercel` o desde la plataforma web, recuerda ir a la configuración de tu proyecto en Vercel (*Settings > Environment Variables*) y registrar estas mismas tres variables (`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`) para que el backend en producción pueda conectarse exitosamente.
