# Tienda Dropshipping Custom (Costo $0/mes)

Este es un proyecto base para una tienda de comercio electrónico de producto único (Landing Page) adaptada para el modelo de **Dropshipping local en Colombia (Pago Contra Entrega)** con integración a **Dropi** y **Meta Ads (Conversión Web + CAPI)**.

## 🚀 Tecnologías Utilizadas
* **Frontend:** HTML5, CSS3, JavaScript (Vite como empaquetador para desarrollo rápido).
* **Backend:** Serverless Functions de Vercel (Node.js).
* **Base de Datos:** Supabase (Base de datos PostgreSQL en la nube gratuita).
* **Servidor y Alojamiento:** Vercel (Capa gratuita).

---

## 📁 Estructura del Proyecto

* `index.html` - Landing page de venta y formulario.
* `style.css` - Diseño visual moderno y optimización móvil.
* `main.js` - Lógica del cliente, selector de ciudades de Colombia y eventos del Píxel.
* `vercel.json` - Configuración de enrutamiento para Vercel.
* `/api`
  * `db.js` - Inicialización de cliente Supabase.
  * `create-order.js` - Procesa formularios, guarda en BD y dispara Conversiones de Meta (Servidor).
  * `export-orders.js` - Exporta pedidos confirmados en formato CSV listo para Dropi.

## 🛠️ Configuración Inicial

### 1. Crear Base de Datos en Supabase
1. Regístrate de forma gratuita en [Supabase.com](https://supabase.com).
2. Crea un proyecto nuevo.
3. Ve a la pestaña de **SQL Editor** en Supabase, haz clic en **New query** y ejecuta el siguiente código para crear la tabla de pedidos:

```sql
create table pedidos (
  id bigint generated by default as identity primary key,
  nombre text not null,
  celular text not null,
  direccion text not null,
  ciudad text not null,
  departamento text not null,
  producto text not null,
  cantidad integer default 1,
  total numeric not null,
  estado_guia text default 'Pendiente', -- Pendiente, Confirmado, Despachado, Cancelado
  id_guia_dropi text,
  sku text,
  fecha timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Habilitar permisos de lectura y escritura (puedes ajustar RLS si lo deseas)
alter table pedidos enable row level security;

create policy "Permitir inserciones públicas"
on pedidos for insert
to public
with check (true);

create policy "Permitir lectura para el rol de servicio"
on pedidos for select
using (true);

create policy "Permitir actualización para el rol de servicio"
on pedidos for update
using (true);
```

### 2. Configurar Variables de Entorno
Crea un archivo llamado `.env` en la raíz del proyecto (basándote en el archivo `.env.example`) y completa las variables de Supabase y Meta Ads:
* `SUPABASE_URL`: La encuentras en Project Settings > API de Supabase.
* `SUPABASE_SERVICE_ROLE_KEY`: La encuentras en Settings > API. **No la expongas en el frontend.**
* `META_PIXEL_ID` y `META_ACCESS_TOKEN`: Necesarios para que funcione el rastreo a nivel de servidor (CAPI).

## 💻 Desarrollo Local

Para correr el proyecto localmente y poder testear tanto el Frontend como las APIs Serverless en tu propio computador, la forma ideal es usando la herramienta oficial **Vercel CLI**:

1. Asegúrate de tener Node.js instalado.
2. Abre la terminal en esta carpeta e instala las dependencias:
   ```bash
   npm install
   ```
3. Instala Vercel de forma global si no lo tienes:
   ```bash
   npm install -g vercel
   ```
4. Corre el proyecto localmente enlazando tus variables del backend:
   ```bash
   vercel dev
   ```
   Esto levantará el servidor local y te proporcionará una URL (normalmente `http://localhost:3000`) donde funcionarán tanto la Landing Page como los endpoints en `/api/*`.

## 📦 Despliegue en Producción (Vercel)

El despliegue a la nube es extremadamente sencillo:
1. Abre tu terminal en la carpeta del proyecto.
2. Escribe el comando:
   ```bash
   vercel
   ```
3. Sigue las instrucciones de la consola.
4. Para pasar a producción oficial con tu dominio, ejecuta:
   ```bash
   vercel --prod
   ```
5. Recuerda añadir las variables de entorno de tu archivo `.env` en la sección de configuraciones de tu proyecto en la plataforma web de Vercel.

## 🚚 Integración con Dropi

1. **Carga Masiva (Opción Inmediata):** 
   Ingresa a `tudominio.com/api/export-orders` desde tu navegador. Se descargará automáticamente un archivo CSV con todos los pedidos que tengan el estado de guía como **"Confirmado"**. Puedes cargar este archivo directamente en tu panel de Dropi en la sección *Pedidos > Importar / Carga Masiva* para generar las guías de envío de forma automática.
2. **Integración API directa (Fase 2):** 
   Cuando soporte de Dropi te apruebe tu `dropi-integration-key` y haga whitelist de tu IP de servidor, puedes activar el consumo directo de sus endpoints en el backend.
